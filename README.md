# 아진산업 연애 시뮬레이션

직장인 인턴 '아진'으로 플레이하는 10일짜리 사내 연애 시뮬레이션 게임.
정규직 전환을 목표로, 4명의 남주 중 한 명과 엔딩을 맞이한다.

## 기술 스택

- **Ren'Py** — 비주얼 노벨 엔진 (스크립트, UI, 세이브 관리)
- **Python** — 게임 로직 (스탯 계산, 엔딩 분기, 아이템 효과)

## 게임 구조

- 10일 × 하루 2행동
- 행동 종류: 업무 / 외모관리 / 휴식 / 상점 / 남주 대화
- 스탯: 매력 / 지능 / 스트레스 / 골드
- 남주 4명 × 호감도 시스템
- 총 11개 엔딩 (해피 4 / 노멀 4 / 특수 1 / 배드 2)

## 실행 방법

1. [Ren'Py SDK](https://www.renpy.org/) 설치
2. 프로젝트 폴더를 Ren'Py 런처에서 열기
3. Launch Project 클릭

## 프로젝트 한계 및 개선 방향

### 1. 실행 환경 의존성
Ren'Py SDK가 없으면 실행 자체가 불가능하다. 링크 하나로 바로 보여줄 수 없어서 접근성이 낮다.
→ React 등 웹 기반으로 재구현하면 배포 링크만으로 데모 가능함.

### 2. DB 연동이 단방향
MariaDB에 게임 데이터를 저장하지만, 읽어서 활용하는 기능이 없다. Ren'Py 내장 세이브도 있고 저장만 하는 DB는 의미가 약하다. flask - DB연동 연습 의의만 있음.
→ 플레이 통계 조회 (엔딩 분포, 인기 남주 순위 등) 기능을 추가하는 방향으로 개선할 수 있다.

### 3. Ren'Py 스택의 확장성 한계
비주얼 노벨 전용 엔진이라 기술 스택을 확장하거나 다른 프로젝트에 응용하기 어렵다.
→ 동일한 게임 로직만 들고와서 웹(React + TypeScript)이나 앱으로 전환하여 재구성 하는게 좋다고 판단됨.

### 4. 결론
만약 개선한다면 renpy를 버리고 새로운 툴을 사용하여 제작하는것이 좋다고 판단됨.
